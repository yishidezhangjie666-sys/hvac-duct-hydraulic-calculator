# 风机 / 水泵选型校核预研

## 1. 预研目标

本预研用于评估在建环工程计算工具箱中新增“风机选型校核”和“水泵选型校核”的可行性。

本阶段只定义功能边界、输入输出、计算口径和后续实现顺序，不直接开发页面和计算代码。

## 2. 项目定位

风机 / 水泵选型校核功能定位为学习、课程设计辅助核算、工程初步校核和个人作品集展示工具。

计算结果不能替代正式工程设计、厂家样本选型或规范校核。

## 3. 是否建议新增

建议新增，但不建议一次性做成复杂设备库。

建议采用“简化选型校核”方式：

- 用户输入系统所需风量 / 风压或流量 / 扬程；
- 用户输入拟选设备参数；
- 工具计算余量；
- 工具给出简化校核状态和复核建议。

## 4. 模块拆分建议

建议拆分为两个独立页签或一个模块内两个页签：

### 4.1 风机选型校核

用于通风风管系统计算完成后，对拟选风机进行风量、风压和功率的初步校核。

### 4.2 水泵选型校核

用于空调水系统计算完成后，对拟选水泵进行流量、扬程和功率的初步校核。

## 5. 风机选型校核设计

### 5.1 建议输入字段

- 方案编号
- 服务系统
- 系统所需风量 L_required（m³/h）
- 系统所需风压 P_required（Pa）
- 风量安全系数 K_L
- 风压安全系数 K_P
- 拟选风机额定风量 L_rated（m³/h）
- 拟选风机全压 P_rated（Pa）
- 电机功率 N_motor（kW）
- 风机效率 η_fan
- 备注

### 5.2 建议输出字段

- 修正后所需风量 L_need（m³/h）
- 修正后所需风压 P_need（Pa）
- 风量余量
- 风压余量
- 风量校核
- 风压校核
- 电机功率参考
- 选型结论
- 复核建议

### 5.3 建议公式

- L_need = L_required × K_L
- P_need = P_required × K_P
- 风量余量 = L_rated / L_need - 1
- 风压余量 = P_rated / P_need - 1
- 轴功率参考：N_ref = L_need × P_need / (3600 × 1000 × η_fan)

说明：

风机功率计算仅作简化估算，实际应结合风机样本、效率曲线、运行点、皮带传动或直联方式、电机效率和安全系数复核。

### 5.4 初步校核状态

建议沿用当前设备选型模块口径：

- 小于 0：不足 / 拟选偏小
- 0 到 20%：合理 / 建议可用
- 20% 到 50%：偏大 / 余量偏大
- 大于 50%：明显偏大
- 输入缺失或非正：需复核

阈值是否最终采用，应在正式开发前再次确认。本预研不修改现有阈值。

## 6. 水泵选型校核设计

### 6.1 建议输入字段

- 方案编号
- 服务系统
- 系统所需流量 G_required（m³/h）
- 系统所需扬程 H_required（m）
- 流量安全系数 K_G
- 扬程安全系数 K_H
- 拟选水泵额定流量 G_rated（m³/h）
- 拟选水泵扬程 H_rated（m）
- 电机功率 N_motor（kW）
- 水泵效率 η_pump
- 备注

### 6.2 建议输出字段

- 修正后所需流量 G_need（m³/h）
- 修正后所需扬程 H_need（m）
- 流量余量
- 扬程余量
- 流量校核
- 扬程校核
- 轴功率参考
- 选型结论
- 复核建议

### 6.3 建议公式

- G_need = G_required × K_G
- H_need = H_required × K_H
- 流量余量 = G_rated / G_need - 1
- 扬程余量 = H_rated / H_need - 1
- 水泵轴功率参考：N_ref = ρgQH / (1000 × η_pump)

其中：

- Q 为 m³/s；
- H 为 m；
- ρ 可默认取 1000 kg/m³；
- g 可取 9.81 m/s²；
- η_pump 为水泵效率。

说明：

水泵功率计算仅作简化估算，实际应结合水泵样本曲线、运行点、汽蚀余量、变频控制、并联运行和电机效率复核。

### 6.4 初步校核状态

建议沿用当前设备选型模块口径：

- 小于 0：不足 / 拟选偏小
- 0 到 20%：合理 / 建议可用
- 20% 到 50%：偏大 / 余量偏大
- 大于 50%：明显偏大
- 输入缺失或非正：需复核

阈值是否最终采用，应在正式开发前再次确认。本预研不修改现有阈值。

## 7. 推荐实现方式

建议后续正式开发时新增一个模块：

```text
modules/fan_pump_selection.py
```

在 `app.py` 中作为第五个模块接入：

```text
风机 / 水泵选型校核
```

模块内部用两个页签：

- 风机选型校核
- 水泵选型校核

## 8. 建议导出能力

后续正式开发时应支持：

- CSV 导出
- Excel 导出
- Word 计算说明书导出

Word 导出应复用现有 `utils/word_report.py`。

校核解释应复用现有 `utils/validation_notes.py`。

## 9. 建议示例数据

风机示例应覆盖：

- 建议可用
- 风量不足
- 风压不足
- 余量偏大
- 明显偏大或需复核

水泵示例应覆盖：

- 建议可用
- 流量不足
- 扬程不足
- 余量偏大
- 明显偏大或需复核

## 10. 暂不计划

本阶段暂不做：

- 厂家设备库
- 自动匹配设备型号
- 风机性能曲线拟合
- 水泵性能曲线拟合
- 并联泵复杂运行分析
- 变频节能分析
- 噪声详细计算
- 汽蚀余量 NPSH 校核
- 规范条文自动判断
- 正式施工图设计选型

## 11. 后续开发顺序建议

建议分三步开发：

### 第一步：计算函数和测试

- 新增 `modules/fan_pump_selection.py`
- 先实现纯计算函数
- 新增 pytest 覆盖建议可用、拟选偏小、余量偏大、需复核

### 第二步：Streamlit 页面

- 接入 app.py
- 增加两个页签
- 增加示例数据
- 增加校核结果解读

### 第三步：导出和文档

- 接入 CSV / Excel / Word 导出
- 更新 README
- 更新 CHANGELOG
- 更新 Roadmap
- 截图和线上检查

## 12. 风险点

- 不能把简化余量判断写成正式规范判断；
- 不能承诺自动完成设备选型；
- 不能忽略设备性能曲线和运行点差异；
- 不能把功率估算作为最终电机选型依据；
- 不能破坏现有四个模块。

## 13. 结论

建议在 `v0.2.0` 后续阶段新增“风机 / 水泵选型校核”模块，但应按简化工程口径推进，并优先完成计算函数、测试和页面原型，再补导出和截图。
