# 后续维护建议

## 当前建议

当前项目已经完成 v0.2.1 发布、归档、展示优化和截图刷新。短期内不建议继续新增大功能，应以稳定、展示和小修为主。

## 优先级 P0：保持稳定

- 不随意改计算公式
- 不随意改校核阈值
- 不随意改导出字段
- 不随意改已发布 tag 和 Release
- 每次修改后必须跑全量测试

## 优先级 P1：小修和展示优化

可做：

- 补充更清晰的截图
- 更新 README 展示顺序
- 补充用户指南中的输入示例
- 优化 Word 导出文本可读性
- 检查线上页面是否和 README 版本一致

## 优先级 P2：谨慎扩展

可评估但不要马上做：

- 新风负荷更细化计算
- 简化负荷估算
- 更多典型工况示例
- 更多专业参数解释
- Word 计算说明书样式细节优化

## 暂不建议

- 厂家设备库
- AI 自动生成计算书
- 大型数据库
- 用户登录
- 商业化收费
- 正式工程设计软件包装
- 大规模重构

## 下一次如果要开发，应先做什么

1. 新建 v0.2.2 Roadmap
2. 明确只做一个小目标
3. 写清楚允许修改范围
4. 先跑基线测试
5. 开发后补测试
6. 不急着打 tag
7. 线上确认后再考虑发布
