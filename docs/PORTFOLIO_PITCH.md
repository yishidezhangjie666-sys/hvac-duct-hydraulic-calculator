# 作品集介绍文案

## 简历版

建环工程计算工具箱：基于 Python + Streamlit 开发的建筑环境与能源应用工程在线计算工具，技术栈包含 pandas、openpyxl、python-docx、pytest 和 GitHub Actions。项目覆盖通风风管、空调水系统、空调末端、冷热源设备和风机 / 水泵选型校核，支持 CSV、Excel 和 Word 计算说明书导出，并保留测试、CI、Release 与发布后检查记录。

## 面试口述版

这个项目来自建环专业课程设计和工程初步核算里的常见场景。很多计算原本分散在手算、Excel 和 Word 文档里，复核和整理都比较麻烦，所以我把通风风管、空调水系统、末端设备、冷热源设备以及风机 / 水泵选型校核整理成一个 Streamlit 在线工具。用户可以录入参数、加载示例数据、查看计算结果和校核建议，也可以导出 CSV、Excel 或 Word 计算说明书。

这个项目重点不只是页面展示，还包括计算模块和导出工具的分层、文档边界说明、pytest 测试、GitHub Actions、版本发布记录和发布后检查记录。它展示的是把专业知识、计算流程和编程工具结合起来的能力，同时也明确说明结果只适合学习、课程设计辅助核算和工程初步校核。

## 项目亮点表达

- 专业方向明确，围绕建环 / 暖通常见计算任务组织功能。
- 功能闭环完整，覆盖输入、计算、校核提示、结果解释和导出。
- 支持 CSV、Excel、Word 多格式导出，便于学习记录和课程设计辅助整理。
- 配置 pytest 和 GitHub Actions，降低后续维护时误改旧功能的风险。
- 已部署在线版本，并保留 Release、Roadmap 和发布后检查记录。
- 项目边界说明清楚，不把简化校核工具包装成正式设计软件。

## 面试可能被问到的问题

**1. 这个工具能不能直接用于正式工程？**

不能。它采用简化工程计算口径，主要用于学习、课程设计辅助核算和工程初步校核。实际工程仍需要结合规范、设计手册、设备样本、厂家曲线和工程经验复核。

**2. 为什么选择 Streamlit？**

Streamlit 适合快速把 Python 计算逻辑做成可交互 Web 工具。这个项目重点是工程计算流程、结果展示和导出，Streamlit 能减少前端开发成本，让更多精力放在计算结构、文档和测试上。

**3. 项目里最核心的模块是什么？**

核心是各类计算模块和导出工具的配合。`modules/` 负责不同专业场景的计算，`utils/` 负责导出、校核说明和 Word 报告生成，让页面、计算和导出逻辑尽量分离。

**4. Word 导出为什么重要？**

课程设计和学习记录往往需要保留输入参数、计算结果、公式说明和免责声明。Word 导出可以把一次计算整理成说明书形式，方便复核、归档和后续修改。

**5. 你怎么保证修改不会破坏已有功能？**

项目有计算函数测试、导出测试、文档链接测试和版本边界测试，本地也有 `scripts/check_project.ps1`。每次改动后会运行编译检查、pytest 和 diff 空白检查，再确认 Git 改动范围。

**6. 后续还能怎么扩展？**

可以继续优化示例数据、截图展示、Word 导出可读性和典型工况说明。大型新模块需要先评估公式依据、输入边界、导出结构和测试覆盖，再小步迭代。
